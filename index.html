QA Process In Agile

Best Practices for an Agile QA Process. When it comes to software development, quality is everything. Quality Assurance (QA) is a systematic process that ensures product and service excellence.

Automation Testing/Tools

Automation testing is a Software testing technique to test and compare the actual outcome with the expected outcome. This can be achieved by writing test scripts or using any automation testing tool

Security Testing

SECURITY TESTING is a type of software testing that intends to uncover vulnerabilities of the system and determine that its data and resources are protected from possible intruders

Selenium

Selenium is an open source tool which is used for automating the tests carried out on web browsers (Web applications are tested using any web browser).

Appium

Appium is an open source test automation framework for use with native, hybrid and mobile web apps. It drives iOS, Android, and Windows apps using the WebDriver protocol

API Testing

API testing is a type of software testing that involves testing (APIs) directly and as part of integration testing to determine if they meet expectations for functionality, reliability, performance, and security

Infra Testing

Infrastructure testing is that part of a test project covering the product risks that relate to the target infrastructure. Typical projects with such product risks include hardware migrations, lifecycle management projects or newly built system deliveries

Testing Techniques

Software testing Techniques help you design better cases. Since exhaustive testing is not possible; Testing Techniques help reduce the number of test cases to be executed while increasing test coverage

BigData Testing

It is defined as testing of Bigdata applications. It is a collection of large datasets that cannot be processed using traditional computing techniques. Testing of these datasets involves various tools, techniques, and frameworks to process
